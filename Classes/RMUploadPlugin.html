RMUploadPlugin Class Reference

Declared in RMUploadPlugin.h

Overview

You uploader bundle principal class.

This class is the go-between for your presets, credentials and configuration view controllers respectively.

The methods which return an NSViewController have assertions performed on the return value, if any of these assertions fail, unless otherwise documented, an exception will be thrown:

  • the view controller's view must not have a superview.

When returning a view controller, make no assumptions about how the view will be used. It may be in a sheet in one version, and a view hierarchy the next. You must not call any methods to deal with closing windows etc. always post the relevant notifications.


A plugin is structured like this:

RMUploadPlugin <->> RMUploadPreset
(RMUploadPlugin <->> RMUploadCredentials)
(RMUploadPreset <<-> RMUploadCredentials)

A plugin owns many presets, and optionally, many credentials.

A preset optionally, has credentials.

credentialsConfigurationViewControllerForCredentials:

This method is required if your preset returns a class from +[RMUploadPreset credentialsClass].

- (RMUploadPresetConfigurationViewController *)credentialsConfigurationViewControllerForCredentials:(RMUploadCredentials *)credentials
Discussion

Used to return an NSViewController whose view allows the user to configure the credentials, such as the username/password and or a token.

You must not configure the plugin<->>credentials relationship in this method, this will be done for you if required.

The RMUploadCredentials object to configure is set as the representedObject property.

additionalMetadataViewControllerForPresetClass:

This method is optional.

- (RMUploadMetadataConfigurationViewController *)additionalMetadataViewControllerForPresetClass:(Class)presetClass
Discussion

If your plugin supports additional file metadata, this view will be placed in your plugin's section of the metadata view. The representedObject will be a KVC container, the contents of which will be passed in the uploadInfo to upload tasks. There is no guarantee that the object you 'configure', and the object passed to your RMUploadTask will be the same, just that they will have the same values you set.

If you do add keys to this container they must be namespaced to your plugin.
